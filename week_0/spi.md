# SPI协议

## SPI介绍

SPI，全称为Serial Peripheral Interface，串行外设接口，由Motorola公司最早推出，是一种广泛使用的同步串行通信接口标准，可在控制器器件与一个或多个外设之间实现高速、全双工数据传输。

SPI 协议允许在一个主机（Master）和一个或多个从机 （Slave）之间建立连接，主机负责生成时钟信号并控制数据传输。SPI 是微控制器和外围IC（如传感器、ADC、DAC、移位寄存器、SRAM等）之间使用最广泛的接口之一。

SPI的四大核心特征：

- **同步传输**: 有时钟信号协调；
- **全双工通信**: 主机与从机之间的发送线和接受线各自独立，同时发送和接收数据；
- **主从架构**: 只能有一个主机，但可以有一个或多个从机
- **高速性能**: 速度的典型范围为1 MHz至100 MHz（4 Mbps至400 Mbps），具体速率取决于器件。

## SPI的分类

#### 4线式SPI

一般默认情况下都是指标准的4线制 Motorola SPI 协议，即SCLK，MOSI，MISO和CS共4根数据线。标准4线式可以实现数据的全双工传输。当只有一个主机和一个从机设备时，只需要一个CS，多个从机需要多个CS。各数据线的介绍：

- **SCLK**: 时钟同步信号，一般由主控来控制; 时钟频率即SPI速率，和SPI模式有关。
- **MOSI**: 即主机输出，从机输入，数据从主机发送到从机。
- **MISO**: 即主机输入，从机输出，数据从从机发送到主机。
- **CS/SS**: 片选，用于选择从机设备，低电平有效，拉高时从机与SPI总线断开连接。

#### 3线式SPI

对于3线式SPI，根据不同的应用场景，主要有以下2种类型：

- 只有3根线：SCLK，MOSI和CS，没有MISO数据线，适用于**单工通讯**。
- 只有3根线：SCLK，SDIO和CS，SDIO作为双向端口，适用于**半双工通讯**。

## SPI特点

#### 1. 主从 (Master-Slave) 模式

与 I2C 类似，SPI 也规定了两个 SPI 设备之间通信必须由主机来控制从机。SPI 一般由 SCLK、CS、MOSI，MISO 四根线组成，有的地方可能是: SCK、SS、SDI、SDO 等名称，含义是一样的。
一个主机可以通过提供 Clock 以及对从机进行片选 (Slave Select) 来控制多个从机, SPI 还规定从机的 Clock 由 主机通过 SCLK 管脚提供给从机, 从机本身不能产生或控制 Clock, 没有 Clock 则从机不能正常工作。

#### 2. 同步 (Synchronous) 模式

在SPI中，主机可以选择时钟极性和时钟相位。

- **CPOL**: 时钟的极性 (clock polarity)；决定 SPI 在空闲时，SCLK 时钟信号是高电平还是低电平。
- **CPHA**: 时钟的相位 (clock phase)；决定数据位传输是从第一个 SCLK 时钟边沿(上升沿/下降沿)开始，还是从第二个边沿开始。

Master 设备会根据将要交换的数据来产生相应的时钟脉冲（Clock Pulse）， 时钟脉冲组成了 SCLK ， 时钟信号通过 CPOL 和 CPHA 控制着两个 SPI 设备间何时数据交换以及何时对接收到的数据进行采样， 来保证数据在两个设备之间是同步传输的。

#### 3. 数据交换 (Data Exchanges)

SPI 协议规定一个 SPI 设备不能在数据通信过程中仅仅只充当一个 Transmitter 或者 Receiver。 在每个时钟周期内，SPI 设备都会发送并接收一个 bit 大小的数据， 相当于该设备有一个 bit 大小的数据被交换了。
SPI总线上的主机必须在通信开始时候配置并生成相应的时钟信号。在每个 SPI 时钟周期内，都会发生全双工数据传输，数据的发送（串行移出到MOSI/SDO总线上）和接收（采样或读入总线(MISO/SDI)上的数据）同时进行。在SPI通信期间，串行时钟沿同步数据的移位和采样。SPI接口允许用户灵活选择时钟的上升沿或下降沿来采样和/或移位数据。

## SPI的工作模式

主机必须根据从机的要求选择 CPOL 和 CPHA。根据 CPOL 和 CPHA 位的选择，可以分为4种模式。**不同的模式，其实就是SCLK空闲时间电平状态和数据采样起点不同。**

| 模式 | CPOL | CPHA | 空闲时SCLK状态 | 采样/移位时刻 |
| ---- | ---- | ---- | ---- | ---- |
| 0 | 0 | 0 | 低电平 | 上升沿采样，下降沿移出 |
| 1 | 0 | 1 | 低电平 | 下降沿采样，上升沿移出 |
| 2 | 1 | 0 | 高电平 | 下降沿采样，上升沿移出 |
| 3 | 1 | 1 | 高电平 | 上升沿采样，下降沿移出 |

- CPOL=0 表示，SCLK 在空闲状态时为0
- CPOL=1 表示，SCLK 在空闲状态时为1
- CPHA=0 表示，在 SCLK 第一个边沿时输入输出数据有效
- CPHA=1 表示，在 SCLK 第二个边沿时输入输出数据有效

这四种模式中，应用最广泛的是**模式0**和**模式3**，大多数SPI器件都同时支持这两种工作模式，但具体采用什么模式需要查看手册。

## 多从机配置

#### 1. 常规模式
通常每个从机都需要一条单独的 CS 线。如果要和特定的从机进行通讯，可以将相应的 CSN 信号线拉低，并保持其他 CSN 信号线的状态为高电平；如果同时将多个 CSN 信号线拉低，则可能会出现乱码，因为从机可能都试图在同一条 MISO 线上传输数据，最终导致接收数据乱码。随着从机数量的增加，来自主机的片选线的数量也增加。这会快速增加主机需要提供的输入和输出数量，并限制可以使用的从机数量。

#### 2. 菊花链模式
在菊花链模式下，所有从机的片选信号连接在一起，数据以串行的方式从一个从机传播到下一个从机。在此配置中，所有从机同时接收同一SPI时钟。来自主机的数据直接送到第一个从机，该从机将数据提供给下一个从机，以此类推直到数据到达目标设备。

<img src="https://www.eetop.cn/uploadfile/2019/0716/20190716042438972.jpg" alt="菊花链模式" title="菊花链模式" width="60%"/>

由于数据是从一个从机传播到下一个从机，所以传输数据所需的时钟周期数与菊花链中的从机位置成比例。
菊花链的最大缺点是因为是信号串行传输，所以一旦数据链路中的某设备发生故障的时候，它下面优先级较低的设备就不可能得到服务了；另一方面，距离主机越远的从机，获得服务的优先级越低，所以需要安排好从机的优先级，并且设置总线检测器。

## 总结

### 与其他协议的比较

SPI 通信通过共享时钟信号实现同步，从而确保数据传输时序的精确性。与集成电路总线 (I2C) 等其他串行协议不同，SPI 没有规定标准的通信格式，因此开发者可根据具体应用需求灵活定制协议参数。下表展示了 SPI、I2C 与通用异步收发传输器 (UART) 之间的差异。

| 功能     | SPI       | I2C  | UART |
| ------ | --------- | ---- | ---- |
| 通信类型   | 同步        | 同步   | 异步   |
| 数据传输模式 | 全双工       | 半双工  | 全双工  |
| 时钟信号   | 是         | 是    | 否    |
| 寻址     | 片选 (CS) 线 | 寻址协议 | 不适用  |
| 速度     | 高         | 中    | 低    |

### 优缺点
- 优点
	- 全双工串行通信, 主从设备可同时收发数据，效率高;
	- 高速数据传输速率, 相比I2C，SPI时钟频率更高;
	- 三态输出的驱动能力强，相对I2C的开漏输出，抗干扰能力强，传输稳定;
	- 灵活的数据传输，不限于8位，它可以是任意大小的字;
	- 简单的硬件结构: 无需地址线（与I2C不同）; 从机使用主机时钟，不需要精密时钟振荡器/晶振（与UART不同）; 不需要收发器（与CAN不同）。

- 缺点
	- 主从架构固定，从设备无法主动发起通信，只能被动响应主设备; 通常仅支持一个主设备;
	- 没有硬件从机应答信号（主机可能在不知情的情况下无处发送）;
	- 需要更多的引脚（与双线的I2C不同），芯片选择线会随着从器件的个数的增加而增加;
	- 没有定义硬件级别的错误检查协议;
	- 与RS-232和CAN总线相比，只能支持非常短的距离。
